Why it's this small

It leans on the platform.

Feather doesn't reimplement components from scratch. The browser ships accessible, keyboard-ready primitives for the overlays, and CSS is plenty for the rest — Feather is the thin React surface on top.

01

Built on primitives

Overlays come from <dialog>, <details>, and the Popover API. Buttons, inputs, and cards are just well-made CSS.

02

Several need zero JS

Accordion, Tabs, Popover, Menu, and Tooltip run on markup and CSS. Nothing to hydrate, nothing to re-render on a toggle.

03

Import only what you use

Every component is its own entry point — feather-kit/button — independently tree-shaken and independently measured.

04

Yours to theme

Everything styles through CSS variables on a single root. Match your design system, or drop in a preset that mirrors a popular kit. Open the theme customizer →

No, really — that's all of it

A whole component.

This is the entire Accordion — props forwarded, children placed, and the <details> element does the rest. Most Feather components read about this short.

export default function Accordion({ summary, open, children, ...rest }) {
  return (
    <details className="feather-accordion" open={open} {...rest}>
      <summary>{summary}</summary>
      <div className="feather-accordion-body">{children}</div>
    </details>
  );
}

No open/close handler, no height-animation hack, no keyboard code — <details> does all of it, and screen readers already understand it. The components that do need JavaScript (Dialog syncs one prop to showModal(); Toast runs a timer) are barely longer.

Drop it in

Install once. Import per component.

One package, one entry point per component. You only ship the bytes for what you actually import.

# install
npm i feather-kit
Button

Variants and sizes, all CSS — no runtime.

import Button from
  "feather-kit/button";

<Button variant="primary">
  Deploy
</Button>
Tabs

A radio group in disguise — no state to wire.

import Tabs from
  "feather-kit/tabs";

<Tabs items={[
  ["One", <A/>],
  ["Two", <B/>],
]} />
Dialog

Modal, focus-trapped, Escape-to-close — all native.

import Dialog from
  "feather-kit/dialog";

<Dialog open={open}
  onClose={close}>
  Saved.
</Dialog>

The honest part

Use the right tool.

Leaning on native primitives means inheriting their limits. Styling the open list of a <select> is still patchy across browsers, popover anchor-positioning is new enough that menus may center on older ones, and <dialog> plus the Popover API exclude the pre-2023 long tail. There's no data grid, no virtualization, no unmount-animation choreography. If you need those, reach for Radix, MUI, or Mantine — and budget the kilobytes. Feather is for the common case: accessible, native-feeling components that don't bloat your bundle. For that, nothing else is this light.
